# anima_pipeline API Reference

## Meta

- Status: Implemented
- Version: v1
- Last Updated: 2026-03-20
- Target release: v1.4.7 (official release)

---

## Overview

anima_pipeline starts an HTTP server at `http://localhost:7860` and exposes the following API. All communication between the frontend (browser UI) and the server uses JSON or binary. The server acts as a middleware layer between the UI and ComfyUI.

---

## Endpoint Reference

### Generation

#### `POST /generate`

Receives input from the UI, optionally generates a prompt via LLM, and sends it to ComfyUI.

**Request Body (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `use_llm` | boolean | Whether to use the LLM |
| `charas` | array | Array of character objects (see below) |
| `scene` | object | Scene / atmosphere settings |
| `params` | object | Generation parameters (seed / steps / cfg / sampler / scheduler) |
| `loras` | array | LoRA slot objects (name / strength) |
| `positive_extra` | object | Positive prompt adjustment settings |
| `negative_extra` | object | Negative prompt adjustment settings |
| `workflow_path` | string | Path or filename of the workflow JSON |
| `image_size` | string | Image size (e.g., `"1024x1024"`) |
| `image_format` | string | Output format (`"png"` or `"webp"`) |

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |
| `llm_prompt` | string | Prompt generated by the LLM (only when LLM is used) |
| `positive_prompt` | string | Positive prompt sent to ComfyUI |
| `negative_prompt` | string | Negative prompt sent to ComfyUI |
| `error` | string | Error message (on failure) |

---

#### `POST /regenerate`

Resends the previous prompts to ComfyUI. Changes in the prompt adjustment fields are applied.

**Request Body (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `positive_extra` | object | Positive prompt adjustments (overrides previous values) |
| `negative_extra` | object | Negative prompt adjustments (overrides previous values) |

**Response:** Same format as `/generate`

### Settings

#### `GET /settings`

Returns the current settings (`settings/pipeline_config.json`).

**Response (JSON):** The contents of `pipeline_config.json` as-is.

---

#### `POST /settings`

Saves settings (writes to `settings/pipeline_config.json`).

**Request Body (JSON):** Object containing the fields to save.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |

---

#### `GET /connection_test`

Tests connectivity to ComfyUI.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |
| `python_version` | string | Python version on the ComfyUI side (on success) |
| `error` | string | Error message (on failure) |

---

#### `GET /llm_connection_test`

Tests connectivity to the LLM API.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |
| `error` | string | Error message (on failure) |

---

### Character Presets

#### `GET /chara_presets`

Returns the list of presets in the `chara/` folder.

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `_ts` | number | Cache-busting timestamp (optional) |

**Response (JSON):** Array of preset objects.

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Preset name (filename without extension) |
| `_thumb_path` | string \| null | Path to the thumbnail if it exists |

---

#### `POST /save_chara_preset`

Saves a character preset.

**Request Body (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Preset name |
| `data` | object | Preset contents |

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |

---

#### `POST /delete_chara_preset`

Deletes a character preset. The associated `.webp` thumbnail is also deleted.

**Request Body (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Name of the preset to delete |

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |

---

#### `POST /chara_preset_thumb`

Generates a thumbnail for a preset from a gallery image and saves it as `chara/<name>.webp`.

> **Added:** v1.4.7

**Request Body (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `preset_name` | string | Name of the preset to associate the thumbnail with |
| `view_url` | string | URL of the source image (ComfyUI view URL or local path) |

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |
| `thumb_path` | string | Path to the saved thumbnail (on success) |
| `error` | string | Error message (on failure) |

---

#### `GET /chara_thumb`

Serves a preset thumbnail image (WebP).

> **Added:** v1.4.7

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `file` | string | Preset name (without extension) |

**Response:** WebP binary (`Content-Type: image/webp`)

---

### LoRA

#### `GET /lora_list`

Scans ComfyUI's LoRA folder and returns the list of available LoRAs.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `loras` | array | Array of LoRA filenames |

---

#### `GET /lora_thumb`

Returns a LoRA thumbnail image. Requires the ComfyUI output folder to be configured.

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `name` | string | LoRA filename |

**Response:** Image binary

---

### Workflows

#### `GET /workflow_list`

Returns the list of JSON files in the `workflows/` folder.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `workflows` | array | Array of workflow filenames |

---

#### `GET /workflow_node_ids`

Auto-detects Positive / Negative / KSampler Node IDs from the specified workflow JSON.

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `file` | string | Workflow filename |

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `positive_node_id` | string | Detected Positive node ID |
| `negative_node_id` | string | Detected Negative node ID |
| `ksampler_node_id` | string | Detected KSampler node ID |

---

### Sessions

#### `POST /save_session`

Saves the current input state as a JSON file.

**Request Body (JSON):** Full session data object.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | `"ok"` or `"error"` |
| `path` | string | Path to the saved file |

---

#### `GET /last_session`

Returns the last auto-saved session (`settings/anima_session_last.json`).

**Response (JSON):** Session data object.

---

### Logging

#### `GET /logs_info`

Returns information about log storage location, file count, and size.

> **Added:** v1.4.7

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `log_dir` | string | Path to the log storage folder |
| `log_level` | string | Current log level (`normal` or `debug`) |
| `retention_days` | number | Log retention period in days |
| `file_count` | number | Number of log files |
| `total_size_bytes` | number | Total size of log files in bytes |

---

#### `GET /logs_zip`

Downloads the log folder as a ZIP archive.

> **Added:** v1.4.7

**Response:** ZIP file (`Content-Type: application/zip`, `Content-Disposition: attachment`)

---

### Gallery

#### `GET /gallery_images`

Returns the gallery image list for the current session.

**Response (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `images` | array | Array of image objects (each includes `view_url`, `positive_prompt`, `negative_prompt`) |

## WebSocket

#### `ws://localhost:7860/ws`

Receives real-time generation progress from ComfyUI.

**Incoming messages (JSON)**

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | `"progress"` / `"done"` / `"error"` |
| `progress` | number | Progress percentage (0–100). Present when `type` is `"progress"`. |
| `message` | string | Status text |

## pipeline_config.json — Key Reference

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `comfyui_url` | string | `http://127.0.0.1:8188` | ComfyUI URL |
| `workflow_path` | string | `image_anima_preview.json` | Workflow JSON path (fallback) |
| `positive_node_id` | string | `11` | Positive prompt node ID |
| `negative_node_id` | string | `12` | Negative prompt node ID |
| `ksampler_node_id` | string | `19` | KSampler node ID |
| `llm_platform` | string | `""` | LLM platform (`lmstudio` / `gemini` / `custom` / empty) |
| `llm_url` | string | `""` | LLM API base URL |
| `llm_token` | string | `""` | LLM API token (masked in logs) |
| `llm_model` | string | `""` | LLM model identifier |
| `output_folder` | string | `""` | Absolute path to ComfyUI output folder |
| `console_lang` | string | `""` | Console display language (follows OS) |
| `log_dir` | string | `logs/` | Log storage folder |
| `log_retention_days` | number | `30` | Log retention period in days |
| `log_level` | string | `normal` | Log level (`normal` / `debug`) |

## HTTP Status Code Reference

| Status | Typical cause |
|--------|--------------|
| 200 | Success |
| 400 | Bad request (invalid parameters, workflow node not found, etc.) |
| 404 | Resource not found (preset, workflow, thumbnail, etc.) |
| 500 | Internal server error (ComfyUI not running, LLM connection failure, etc.) |
