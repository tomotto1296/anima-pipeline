## Goal / 目的

毎回同じキャラを手動で設定するのが面倒で、新しいキャラを試す機会が少ない。髪色・瞳色・衣装・体型などの属性をランダムに組み合わせて新規プリセットを自動生成し、試し生成の起点を作る。

---

## Scope

- **In Scope:**
  - `ui_options.json` の既存選択肢（髪色・瞳色・衣装・体型・性別・年齢等）からランダムに属性を選んでキャラスロットに適用する
  - 適用した内容をプリセットとして保存するオプション（保存名を入力して `chara/` に保存）
  - ランダム生成対象の属性を絞れるオプション（フェーズ2）

- **Out of Scope:**
  - LLMを使ったキャラ名・作品名の自動生成
  - 既存プリセットからのランダムロード（GEN-9の今日の気分ボタンで対応）
  - Danbooru Wikiを使ったプリセット自動生成（既存の `/generate_preset` で対応済み）

---

## User Story

> As a ユーザー, I want 属性をランダムに組み合わせた新規キャラをワンクリックで作れる, so that 手動設定なしに新しいキャラの試し生成を始められる.

---

## Requirements / 要件

1. `▶ キャラクター` セクション内のプリセット一覧付近に「🎲 ランダム生成」ボタンを追加する。
2. クリックで以下の属性をランダムに選択してキャラスロット（デフォルト: スロット1）に適用する:
   - 性別（`ui_options.json` の `gender_options`）
   - 年齢（`age_options`）
   - 髪型（`ui_options.json` の選択肢）
   - 髪色（`ui_options.json` の選択肢）
   - 瞳色（`ui_options.json` の選択肢）
   - 衣装カテゴリ（上下 / 全身 等）
3. 適用後に「このキャラを保存しますか？」のダイアログを表示し、名前を入力して `chara/` にプリセット保存できる。保存をスキップしてもスロットへの適用は維持される。
4. 適用先スロットは「更新先」セレクトで選択できる（既存の更新先セレクトを流用する）。
5. 適用後に適用した属性のサマリーをSTATUSまたはトーストで短時間表示する。

## UI/UX

### 表示場所 / Screen

`▶ キャラクター` セクション内、プリセット一覧の「＋ キャラ追加」ボタン付近

### 変更点 / Changes

```
[プリセット一覧] [＋ キャラ追加] [🎲 ランダム生成]   <- ボタン追加
```

### Labels (JA / EN)

| JA | EN |
|----|-----|
| ランダム生成 | Random Generate |
| このキャラを保存しますか？ | Save this character? |

## API / Data

新規エンドポイントなし。ランダム選択はフロントエンドのJSで実装し、保存は既存の `POST /save_chara_preset` を使用する。

### 保存先 / Storage

| 種別 | パス |
|------|------|
| キャラプリセット（保存時） | `chara/<n>.json` |

## 既存機能との互換性

- **既存プリセット（`chara/*.json`）:** 保存時のみ追加。既存プリセットへの影響なし。
- **既存の `/generate_preset`（Danbooru Wiki + LLM）:** 別機能として共存する。
- **`pipeline_config.json` スキーマ変更の有無:** なし。

---

## Error Handling / エラー処理

| 想定エラー | 表示文言（JA） | 表示文言（EN） | ログ出力 |
|-----------|-------------|-------------|---------|
| プリセット保存失敗 | プリセットの保存に失敗しました | Failed to save preset | `[INPUT-1]` |
| `ui_options.json` の選択肢が空 | ランダム生成に必要な選択肢が見つかりません | No options available for random generation | `[INPUT-1]` |

## Acceptance Criteria / 完了基準

1. ランダム生成ボタンクリックでキャラ属性がランダムにスロットへ適用される
2. 適用後に保存ダイアログが表示され、保存または스킵できる
3. 保存した場合 `chara/` にプリセットが生成される
4. 既存のプリセット保存・読込・セッション保存が壊れない

## Test Plan / テスト計画

### Manual

- ランダム生成ボタンクリック → 属性がスロットに適用されること
- 保存ダイアログで名前を入力 → `chara/` にJSONが生成されること
- 保存ダイアログをスキップ → スロットへの適用は維持されること
- 「更新先」でスロット2を選択してランダム生成 → スロット2に適用されること

### Edge Cases

- `ui_options.json` の特定選択肢が空の場合（その属性をスキップするか確認）
- 保存時に同名プリセットが存在する場合（上書き確認ダイアログを表示）

### Regression

- 既存の「＋ キャラ追加」ボタンが正常に動作するか
- 既存のDanbooru Wiki + LLMプリセット自動生成が正常に動作するか
- プリセット一覧UIが崩れていないか
