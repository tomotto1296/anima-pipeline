## Goal / 目的

再生成時にプロンプトをどう変えたか確認できず、試行錯誤の記録が残らない。前回生成と今回生成のプロンプトの差分をハイライト表示することで、変更箇所を一目で把握できるようにする。

---

## Scope

- **In Scope:**
  - 最後に生成したプロンプト（ポジティブ）と現在のプロンプトの差分表示
  - 追加されたトークンを緑、削除されたトークンを赤でハイライト
  - 差分ビューはトグル式で表示・非表示を切り替えられる
  - ポジティブプロンプトのみ対応（ネガティブはOut of Scope）

- **Out of Scope:**
  - ネガティブプロンプトの差分表示
  - 複数世代前との比較
  - 差分の保存・履歴管理（OUTPUT-3の履歴DBと別軸）

---

## User Story

> As a ユーザー, I want 前回との変更箇所がひと目でわかる, so that 試行錯誤の方向性を確認しながら生成を繰り返せる.

---

## Requirements / 要件

1. 「送信POSITIVEプロンプト」表示エリアの近くに「差分を表示」トグルボタンを追加する。
2. トグルON時、前回生成のプロンプトと現在のプロンプトの差分をトークン単位で表示する。
3. 追加トークンは緑背景、削除トークンは赤背景（打ち消し線付き）でハイライトする。
4. 前回プロンプトはブラウザのメモリ上に保持する（ページリロードでリセット）。
5. 初回生成時（前回プロンプトなし）は差分ビューを表示しない。

---

## UI/UX

### 表示場所 / Screen

「送信POSITIVEプロンプト」表示エリアの下

### 変更点 / Changes

```
送信POSITIVEプロンプト: [_______________]
[差分を表示 ▼]                              <- 新規追加（トグル）

差分ビュー（トグルON時）:
  normal_quality [-worst_quality] [+masterpiece] 1girl ...
  赤=削除 / 緑=追加
```

### Labels (JA / EN)

| JA | EN |
|----|-----|
| 差分を表示 | Show diff |
| 差分を非表示 | Hide diff |
| 前回なし | No previous prompt |
| 変更なし | No changes |

## API / Data

新規エンドポイントなし。差分計算はフロントエンドのJSで実装する。

### 保存先 / Storage

前回プロンプトはJSの変数として保持（永続化なし）。

## Error Handling / エラー処理

差分計算はフロントエンドのみで完結するためAPIエラーなし。差分計算失敗時は差分ビューを非表示にする。

---

## Acceptance Criteria / 完了基準

1. 生成後に「差分を表示」ボタンが現れ、クリックで差分が表示される
2. 追加トークンが緑、削除トークンが赤でハイライトされる
3. 初回生成時は差分ビューが表示されない
4. 既存のプロンプト表示・生成フローが壊れない

## Test Plan / テスト計画

### Manual

- 1回目生成 → 差分ボタンが非表示（または「前回なし」表示）
- 2回目生成（プロンプト変更あり）→ 差分ボタンが現れ、クリックで差分が表示される
- 追加トークンが緑、削除トークンが赤で表示されること
- 変更なしで再生成 → 「変更なし」と表示されること

### Edge Cases

- プロンプトが空の場合
- 前回と今回が完全に同一の場合
- 非常に長いプロンプトの場合（表示が崩れないか）

### Regression

- 既存のプロンプト表示エリアが崩れていないか
- 生成フローが正常に動作するか
