# AnimaPipeline 機能追加案リスト

最終更新: 2026-03-21  
v1.4.7以降の機能追加候補。優先度はCodex・Claude・各AI案を統合して決定。

## 優先度定義

| 優先度 | 意味 |
|--------|------|
| MUST | 基盤・早期必須（後回しにすると他機能がやり直しになる） |
| HIGH | 効果大・早めに入れたい |
| MID | あると嬉しい |
| LOW | 余裕があれば |
| SKIP | 見送り推奨 |

## カテゴリ定義

| カテゴリ | 対象フェーズ |
|----------|------------|
| SETUP | 初回・導入 |
| INPUT | 生成前の入力・プリセット管理 |
| GEN | 生成・実行 |
| OUTPUT | 生成後・管理・再現 |
| SHARE | 共有・発信 |
| UI | 見た目・操作性 |

## 推奨実装順（エンジニア視点）

```
フェーズ1（基盤）
  OUTPUT-4（メタデータ基盤）
    └─ OUTPUT-3（履歴DB）
         └─ INPUT-4（プリセット構造整理）
              └─ INPUT-5（ネガティブプリセット）
                   └─ OUTPUT-8（セッション複数保存）

フェーズ2（導入体験・日常利用）
  SETUP-2（自己診断UI）     ← MUSTに昇格、早めに
  INPUT-6（LoRA管理強化）   ← MUSTに昇格、早めに
  INPUT-1（ランダムプリセット）
  SHARE-1（プリセット共有zip）← HIGHに昇格

フェーズ3（量産・比較）
  GEN-1（一括生成）
  GEN-2（生成キュー）
  GEN-3（比較生成）

フェーズ4（再現・高度化）
  OUTPUT-1（画像→プロンプト再現）← MIDに調整、基盤後に
  OUTPUT-2（ワークフロー再現補助）
  OUTPUT-5（LLM評価・振り分け）
  OUTPUT-6（自動Danbooruタグ）

フェーズ5（最後）
  GEN-8（動画）      ← 検討のみ
  OUTPUT-10（LoRA自動DL）← 検討のみ
```

> 順番を気にしなくていい独立案: UI-5（ダークモード）、UI-6（QRコード）、SETUP-3（Gemini残り回数）、INPUT-11（Wikiヒット率）
