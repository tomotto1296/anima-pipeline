## 優先度定義

| 優先度 | 意味 |
|--------|------|
| MUST | 基盤・早期必須（後回しにすると他機能がやり直しになる） |
| HIGH | 効果大・早めに入れたい |
| MID | あると嬉しい |
| LOW | 余裕があれば |
| SKIP | 見送り推奨 |

## カテゴリ定義

| カテゴリ | 対象フェーズ |
|----------|------------|
| SETUP | 初回・導入 |
| INPUT | 生成前の入力・プリセット管理 |
| GEN | 生成・実行 |
| OUTPUT | 生成後・管理・再現 |
| SHARE | 共有・発信 |
| UI | 見た目・操作性 |

## 推奨実装順（エンジニア視点）

```
v1.4.8（基盤フェーズ）
  OUTPUT-4（メタデータ基盤）← ✅ 完了（v1.4.718）
    └─ OUTPUT-3（履歴DB）← ✅ 完了（v1.4.730）
         └─ INPUT-4（プリセット構造整理）← 次
              ├─ INPUT-12（キャラ名・作品名 日英欄分離）← INPUT-4と同タイミングでスキーマ変更を1回にまとめる
              └─ INPUT-5（ネガティブプリセット）
                   └─ OUTPUT-8（セッション複数保存）
  SETUP-2（自己診断UI）
  REFACTOR-1（モジュール分割）← v1.4.8の機能追加が一通り終わった後に実施

v1.4.9（日常利用強化フェーズ）
  INPUT-6（LoRA管理強化）
  INPUT-1（ランダムプリセット）
  SHARE-1（プリセット共有zip）
  OUTPUT-9（プロンプト差分ビューア）
  GEN-9（今日の気分ボタン）

v1.5.0（量産・目玉フェーズ）
  GEN-1（一括生成）← メインの目玉
  GEN-2（生成キュー）
  GEN-3（比較生成）
  OUTPUT-1（画像→プロンプト再現）← 基盤が揃ったタイミングで

v1.5.x〜（高度化フェーズ）
  OUTPUT-2（ワークフロー再現補助）
  OUTPUT-5（LLM評価・振り分け）
  OUTPUT-6（自動Danbooruタグ）
  GEN-5（キャラシート自動生成→一括生成）
  その他MID案を順次
```

> 順番を気にしなくていい独立案: UI-5（ダークモード）、UI-6（QRコード）、SETUP-3（Gemini残り回数）、INPUT-11（Wikiヒット率）
