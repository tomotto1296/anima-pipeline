# anima_pipeline 使い方ガイド

## 概要

ブラウザUIでキャラ・シーン情報を入力 → LLM（オプション）でDanbooruプロンプトを自動生成 → ComfyUI Animaワークフローへ自動送信するパイプラインツールです。

---

## 必要なもの

| ツール | バージョン | 用途 |
|--------|-----------|------|
| Python | 3.10以上 | サーバー起動 |
| requests | （pip install requests） | HTTP通信ライブラリ（初回自動インストール） |
| ComfyUI | 0.16.4以上 | 画像生成 |
| Animaワークフロー | — | ComfyUI用ワークフロー |
| LLMサーバー（LM Studio等） | — | プロンプト自動生成（**オプション**） |

---

## ファイル構成

配布セット：

```
anima_pipeline/
  anima_pipeline.py              ← メインスクリプト
  start_anima_pipeline.bat       ← Windows起動用バッチ
  anima_pipeline_guide.md        ← このファイル
  image_anima_preview.json       ← Animaワークフロー（ComfyUI用）※後述
  chara/
    000_default.json             ← キャラプリセット初期値（全項目空）
  workflows/                     ← ワークフローJSONを置くフォルダ（自動作成）
    image_anima_preview.json       ← LoRAなし標準ワークフロー
    image_anima_preview_Lora4.json ← LoRA×4対応ワークフロー
  settings/
    pipeline_config.json         ← 初期設定（ComfyUI URL・Node ID等）※APIキー入力後はgit管理除外推奨
    pipeline_config.default.json ← 初期値テンプレート（git管理用）
    ui_options.json              ← UIボタン選択肢の定義（編集可）
    llm_system_prompt.txt        ← LLMへのシステムプロンプト（編集可）
    preset_gen_prompt.txt       ← キャラプリセット自動生成用プロンプト（編集可）
```

初回起動後に自動生成されるファイル（配布不要）：

```
  settings/
    pipeline_config.json         ← 設定変更後の上書き保存先（配布時の初期値から変更される）
    extra_tags.json              ← ポジティブExtraタグのプリセット
    extra_tags_negative.json     ← ネガティブExtraタグのプリセット
    style_tags.json              ← スタイルタグのプリセット
    anima_session_last.json      ← 最終セッションの自動保存
```

> **`image_anima_preview.json` について**  
> このファイルはリポジトリに含まれていません。以下の手順でご自身の環境から取得して `anima_pipeline.py` と同じフォルダに配置してください。
>
> 1. ComfyUIを起動
> 2. 右上のメニュー（三本線）→「Browse Templates」を開く
> 3. **Anima**テンプレートを選択して読み込む
> 4. メニュー →「Save (API Format)」で `image_anima_preview.json` として保存
> 5. 保存したファイルを `anima_pipeline.py` と同じフォルダに配置

## 起動方法

### Windows
`start_anima_pipeline.bat` をダブルクリック

> **batファイルがVS Code等で開いてしまう場合：** 右クリック →「プログラムから開く」→「コマンドプロンプト」を選択するか、右クリック →「管理者として実行」してください。
>
> 初回起動時に `requests` ライブラリが未インストールの場合、自動でインストールします。

### 手動起動
```bash
python anima_pipeline.py
```

起動後、ブラウザで `http://localhost:7860` を開く。

### スマホ・タブレットからアクセスする（v1.4.1〜）

PCと同じWi-Fi環境にあるスマホ・タブレットからアクセスできます。

**手順：**

1. PCで `ipconfig`（Windows）を実行し、**IPv4アドレス**を確認する（例: `192.168.1.10`）
2. ComfyUIを以下の引数で起動する（`--listen` と `--enable-cors-header` が必要）

```
.\python_embeded\python.exe -s ComfyUI\main.py --windows-standalone-build --lowvram --listen --enable-cors-header
```

3. anima_pipelineを通常通り起動する
4. スマホのブラウザで `http://192.168.1.10:7860` を開く（IPアドレスは手順1で確認したもの）

> **注意：** アドレスバーには必ず `http://` から始めて入力してください。`https://` にするとSSLエラーが出ます。
>
> LoRAサムネイルはComfyUI outputフォルダの設定が必要です（設定パネル参照）。

## 初回設定

画面上部の「▶ 設定」を開いて確認・入力します。設定は「💾 設定を保存」で `settings/pipeline_config.json` に保存されます。

> **②③④はAnima標準ワークフロー向けの初期値が最初から入っています。** 別のワークフローを使う場合のみ変更してください。

> **注意：** 初回起動時はプラットフォームが「なし」の状態です。LLMを使用する場合は設定パネルでプラットフォームを選択し、「💾 設定を保存」を押してください。なお生成開始ボタンを押すたびに設定は自動保存されます。

### 🔴 必須

| 番号 | 項目 | 初期値 | 説明 |
|------|------|--------|------|
| ① | ワークフローJSONパス（フォールバック） | `image_anima_preview.json` | `workflows/`フォルダで未選択の場合に使用。フルパスまたはファイル名で指定 |
|   | ワークフロー選択（優先） | — | `workflows/`フォルダ内のJSONをドロップダウンで選択。🔄で再スキャン。選択するとこちらが優先。**選択時にNode IDを自動検出して設定反映**（ControlNet等が挟まる場合は手動確認推奨） |
| ② | Positive Node ID | `11` | Animaワークフロー標準。別ワークフローは要確認 |
| ③ | Negative Node ID | `12` | Animaワークフロー標準。別ワークフローは要確認 |
| ④ | KSampler Node ID | `19` | KSamplerノードのID。seed/steps/cfg/sampler注入先 |
| ⑤ | ComfyUI URL | `http://127.0.0.1:8188` | 変更している場合のみ修正 |

### 🟠 LLMを使うなら必須

⑥ **LLMプラットフォーム**を選択するとURL・Token・モデル名の入力欄が開きます。

| プラットフォーム | 説明 |
|------|------|
| **なし** | LLM機能を使わない。テキスト直接入力のみ |
| **LM Studio** | ローカルLLM。無料・無制限だが高スペックPCが必要 |
| **Gemini** | Google Gemini API。無料枠あり（要APIキー、クレジットカード不要） |
| **カスタム** | 上記以外のOpenAI互換APIを手動設定 |

> **LLMプラットフォームの要件：** 使用するサービスがOpenAI互換API（`/v1/chat/completions`形式）に対応している必要があります。各サービスのAPI利用規約・料金・レート制限は各自でご確認ください。

| 番号 | 項目 | 説明 |
|------|------|------|
| ⑤-1 | LLM URL | APIのベースURL |
| ⑤-2 | LLM API Token | 認証が必要な場合のみ入力。ローカル（LM Studio）なら空欄でOK |
| ⑤-3 | LLMモデル名 | 使用するモデルの識別子 |

### 🟢 任意

| 番号 | 項目 | 説明 |
|------|------|------|
| ⑦ | LLMツール統合 | Danbooru RAG / Danbooru API / DuckDuckGoのON/OFF（LM Studio専用） |
| ⑧ | ComfyUI outputフォルダ | WebP変換用。**絶対パスで入力推奨**（例: `D:\ComfyUI_Portable\ComfyUI_windows_portable\ComfyUI\output`）。空欄の場合は自動推定を試みるが、ワークフローJSONをファイル名のみで指定している場合は失敗することがある |

### 🔧 デバッグ用

設定パネル下部に以下のデバッグ機能があります。

| 機能 | 説明 |
|------|------|
| 🔌 ComfyUI 接続テスト | ComfyUIへの接続確認。成功時はPythonバージョンを表示 |
| 🤖 LLM 接続テスト | LLM APIへの接続確認。URLとAPIキーが正しいか確認できる |
| 📋 キャラプリセット削除 | キャラプリセットの削除（誤操作防止のため設定パネル内に配置） |

## 💡 おすすめの使い方

**英語でタグを直接入力してLLMを介さずに送信するのが一番速いです。**

1. 「LLMを使用する」チェックをOFF
2. 詳細フィールドのLLMチェックをすべてOFF
3. 英語タグを直接入力して「▶ 生成開始」

LLMが便利なのは「キャラ名・作品名から自動でDanbooruタグを引っ張ってきたい」ときです。

### 入力欄の色分け

| 色 | 意味 |
|----|------|
| 🔵 水色 | 日本語OK（LLMが解釈して英語タグ化します） |
| 🟢 緑色 | 英語専用（Danbooruタグや英文を直接入力） |

## 別のワークフローで動かしたいとき

ComfyUIのAnima以外のワークフローや、別バリアントのワークフローで使う場合：

1. ComfyUIでワークフローを開き、**「Save (API Format)」でJSONを保存**
2. 設定の「① ワークフローJSONパス」を新しいJSONのパスに変更
3. 「② Positive Node ID」「③ Negative Node ID」をそのワークフローのノードIDに合わせて更新
4. 「💾 設定を保存」

> Node IDの確認方法：ComfyUIでプロンプトノードを右クリック →「Properties」のNode IDを確認

## 基本的な使い方

### A. キャラ情報を入力

**キャラプリセット**（各キャラブロックの上部）
- よく使うキャラの設定をプリセットとして保存・呼び出しできます
- 保存場所：`anima_pipeline/chara/` フォルダ（1プリセット1ファイル）
- 「保存」：そのキャラ枠の設定をプリセット名を入力して保存
- 「読込」：選択したプリセットをそのキャラ枠に流し込む
- 「🔍」：キャラ名・作品名からDanbooru Wiki + LLMでプリセットを**自動生成**して読込（LLM設定が必要）
  - 自動生成用プロンプトは `settings/preset_gen_prompt.txt` で編集可能
  - Wikiにキャラが存在しない場合はLLMの知識のみで生成（有名キャラなら概ね正確）
  - 生成後に手動で修正→保存し直す運用推奨
- **削除は「▶ 設定」パネル内の「キャラプリセット削除」から行います**（誤操作防止）
- `chara/000_default.json` が初期プリセットとして同梱されています（全項目空）
- **LLMチェックの状態はプリセットに含まれません**。読み込み後に各自で設定してください

**キャラ名・作品名**（水色欄・日本語OK）
- LLMあり：キャラ名と作品名を入力するとLLMがDanbooruで検索してタグを生成します
- LLMなし：キャラ名・作品名を `name_(series)` 形式に自動変換してプロンプトに追加します（英語入力推奨）
- **オリジナルキャラ**の場合は「オリジナル」ボタンを押す → 作品名欄が無効化され、LLMチェックが全部OFFになります
- キャラブロックの「＋」ボタンで複数キャラを同時設定可能

**性別・年齢**（`ui_options.json`で選択肢を変更可能）
- LLMチェックON: LLMが判断
- LLMチェックOFF: 選択値を直接Danbooruタグに変換（`1girl` / `1boy` など）

### B. 詳細を入力（任意）

「＋ 詳細」ボタンで展開します。

| フィールド | 説明 |
|-----------|------|
| ① 髪型 | 全体・前・後ろのグループから単一選択。自由入力欄は**日本語OK** |
| ② 髪色 | ドロップダウンから選択（25色）。その他は自由入力欄に日本語で記入 |
| ③ 目の状態 | 開閉（単一選択）・向き・状態のグループ |
| ④ 瞳の色 | ドロップダウンから選択（20色）。オッドアイは「odd」ボタンで左右個別設定 |
| ⑤ 口 | 複数選択 |
| ⑥ 表情 | 複数選択 |
| ⑦ 肌の色 | ドロップダウンから選択（色付き表示）。異色肌は自由入力欄に日本語で記入 |
| ⑧ 衣装 | カテゴリ（全裸/半裸/全身/上下）を押すと色・種類が展開。上下は上半身・下半身を個別設定 |
| ⑨ バスト | 女性・不明キャラのみ表示 |
| ⑩〜⑫ | 背丈・体型・脚 |
| ⑬ 付属 | 耳・尻尾・翼（単一選択）、アクセサリー（複数選択） |
| ⑭ エフェクト | 汗/涙/オーラ/流血など複数選択 |
| ⑮ 動作・ポーズ | 姿勢/動作/腕・手/視線のグループから複数選択。自由入力は**英語タグ直接** |
| ⑯ 持ち物 | 「持つ」をONにすると展開。自由入力は**英語タグ直接** |
| ⑰〜⑲ | 画面上下・左右・カメラ |

**LLMチェックについて**
- ON → フィールドの値をLLMに渡してタグ生成を任せる
- OFF → 選択値をそのままDanbooruタグに直接変換（高速・確実）

### C. シーン・雰囲気を設定（任意）

キャラクターの下に独立したブロックとして表示されます。

| 項目 | 選択肢 |
|------|--------|
| 世界観 | 日常 / 和風 / 西洋 / 中華 / ファンタジー / SF / 廃墟 |
| 場所 | 屋外・屋内・特殊のカテゴリから選択、または自由入力（水色欄・**日本語OK**） |
| 時間帯 | 朝 / 昼 / 夕 / 夜 |
| 天気 | 晴 / 曇 / 雨 / 雪 |
| 補足メモ（日本語） | LLMへの雰囲気ヒント。**1回目の生成のみ**有効 |

### D. 生成パラメータ（任意）

「▶ 生成パラメータ」セクションでKSamplerの設定を変更できます。

| 項目 | 説明 |
|------|------|
| Seed（シード値） | ランダム / 固定 / 連番 から選択。🎲ボタンでランダム値を生成 |
| Steps | サンプリングステップ数（デフォルト: 30） |
| CFG | CFGスケール（デフォルト: 4） |
| Sampler | サンプラー名（ドロップダウンから選択） |
| Scheduler | スケジューラー名（ドロップダウンから選択） |

> Sampler・Schedulerの選択肢は `settings/ui_options.json` の `sampler_options` / `scheduler_options` で変更できます。

### E. LoRA（任意）

「▶ LoRA」セクションでLoRAの選択と強度を設定できます。

- 🔄 LoRA一覧取得ボタンでComfyUIのLoRAフォルダをスキャン
- スロットは最大4つ。各スロットでLoRAファイルと強度（0〜2）を設定
- 空のスロットはスキップされます
- ワークフロー内の `LoraLoader` ノードを順番に書き換えて注入
- LoRAを使う場合はLoraLoaderノードを含むワークフロー（例：`image_anima_preview_Lora4.json`）を選択してください
- スロット数よりLoraLoaderノードが多い場合は、余ったノードをバイパス（自動削除）して送信します
- LoraLoaderノードの数がスロット上限です（`image_anima_preview_Lora4.json`はLoRA×4まで対応）

> セッション保存・復元にも対応しています。

### F. ポジティブプロンプト調整・追加（任意）

「▶ プロンプト調整・追加」欄。**再画像生成にも反映されます。**

| 番号 | 項目 | 説明 |
|------|------|------|
| ① | 期間タグ | Animaモデル向け最適化タグ（先頭挿入） |
| ② | 品質タグ | masterpiece / score_9 など |
| ③ | メタタグ | highres / absurdres など |
| ④ | 安全タグ | safe / nsfw など（単一選択） |
| ⑤ | スタイル | @アーティスト名。**プリセット登録可能**。右クリックで削除 |
| ⑥ | Extraタグ | プリセット登録可能。右クリックで削除。`settings/extra_tags.json`に永続保存 |
| ⑦ | 追記文（英語） | プロンプト末尾に直接追加する英文（緑色欄） |

### G. ネガティブプロンプト調整（任意）

「ネガティブプロンプト調整」欄（赤枠）。

| 番号 | 項目 | 備考 |
|------|------|------|
| ① | 期間タグ | ポジティブと共通 |
| ② | 品質タグ | 人間ベース: NORMAL/LOW/WORST（デフォルトON）、Pony（デフォルトOFF） |
| ③ | メタタグ | デフォルトOFF |
| ④ | 安全タグ | ポジティブと共通 |
| ⑤ | スタイル | ポジティブと共通 |
| ⑥ | Extraタグ | ネガティブ専用プリセット。`settings/extra_tags_negative.json`に永続保存 |
| ⑦ | 追記文（英語） | ネガティブプロンプト末尾に直接追加 |

### H. 生成

**LLMを使用する場合**
1. 「LLMを使用する」チェックがONであることを確認
2. 「▶ 生成開始」をクリック（またはCtrl+Enter）
3. LLMがプロンプトを生成 → ComfyUIに送信 → 画像生成

**LLMをスキップする場合**
1. 「LLMを使用する」チェックをOFF
2. 「▶ 生成開始」をクリック
3. LLMチェックの状態に関わらず全フィールドが直接Danbooruタグとして送られます（速い）
   - キャラ名・作品名は `name_(series)` 形式に自動変換
   - シーン情報（場所・時間帯・天気等）も自動で含まれます

生成完了後、処理状況欄に以下が表示されます：
- **LLM生成ポジティブプロンプト** — LLMが生成した内容
- **ComfyUI 送信ポジティブプロンプト** — 最終的にComfyUIに送った内容（コピーボタン付き）
- **ComfyUI 送信ネガティブプロンプト** — ネガティブ側の送信内容（赤文字・コピーボタン付き）

## ギャラリー（生成履歴）

生成完了後、ページ下部に「📷 生成履歴（このセッション）」が表示されます。

- サムネイルをクリックすると画像を拡大表示
- 拡大表示から送信ポジティブ・ネガティブプロンプトを確認
- 「↺ プロンプト再利用」で再画像生成モードに設定
- 「📁 フォルダを開く」で画像の保存フォルダをエクスプローラーで開く（要: outputフォルダの設定）
- 「クリア」ボタンでセッション内の履歴を削除

> ギャラリーはセッション内のみ保持されます。再起動するとリセットされます。

## 再画像生成

「↻ 再画像生成」ボタンで、前回と同じプロンプトをComfyUIに再送信します。  
ポジティブ・ネガティブのプロンプト調整欄の変更は再生成にも反映されます。

## スタイルタグのプリセット管理

絵師スタイルを複数キャラ・複数セッションで使いまわせます。

1. スタイル入力欄（緑色）に`takeuchi naoko`などを入力して「追加」
2. プリセットボタンとして登録される（`settings/style_tags.json`に保存）
3. 次回以降はプリセットをクリックするだけでON/OFF切り替え
4. 右クリックでプリセットから削除

## 画像サイズのカスタマイズ

`settings/ui_options.json` の `image_size_presets` を編集すると、ドロップダウンの選択肢を変更できます。

```json
"image_size_presets": [
  {"v": "1024x1024", "label": "1024 × 1024　正方形"},
  {"v": "1920x1080", "label": "1920 × 1080　フルHD横"}
]
```

## WebP出力について

**注意：** WebP形式を選択した場合、ComfyUIが生成したPNGファイルを直後に自動でWebPに変換します。  
**変換後はPNG内に埋め込まれたプロンプト・ワークフロー情報は破棄されます。**  
ワークフロー情報を残したい場合はPNG形式を使用してください。

## トラブルシューティング

| 症状 | 確認事項 |
|------|---------|
| batファイルがVS Code等で開く | 右クリック →「プログラムから開く」→「コマンドプロンプト」を選択 |
| `No module named 'requests'` エラー | `pip install requests` を実行するか、batファイルを右クリック →「管理者として実行」 |
| ComfyUIに送信されない | ComfyUI URLとワークフローJSONパスを確認 |
| 自前WFで「Positiveノード XXX が存在しません」 | Save (API Format)で出力したJSONを使用しているか確認。v1.4.3以降で自動対応済み |
| Negative Promptが反映されない | 設定の「③ Negative Node ID」が正しく設定されているか確認 |
| LLMが応答しない | LLMサーバーが起動しているか確認。モデル名・URLが正しいか確認 |
| 画像が表示されない / WebP変換されない | ComfyUI outputフォルダに絶対パスを入力して保存。ワークフローJSONをファイル名のみで指定している場合は空欄での自動推定が失敗する |
| MCPツールが動かない | LLMサーバー側のMCPプラグインを確認。または設定欄でツール統合をOFFに |
| セッション復元がずれる | 古いバージョンのセッションファイルは自動で新フォーマットに変換されます。それでもずれる場合はリロードで解消 |
| LoRAサムネが表示されない | ComfyUI outputフォルダを設定パネルで設定してください。また ComfyUI の起動引数に `--listen --enable-cors-header` が必要です |
| スマホからギャラリー画像が表示されない | ComfyUIの起動引数に `--listen` が必要です |
| LLMが思考過程を長々と出力する / 応答が遅い | 下記「Qwen3系モデルの思考出力について」を参照 |
| 生成画像が真っ白 / ノイズまみれ | LLMの思考テキストがプロンプトに混入している可能性。下記「Qwen3系モデルの思考出力について」を参照 |
| UI言語が切り替わらない / 一部が切り替わらない | ページをリロードする。動的ラベルは切替後に再描画されるため、生成動作後に確認する |
| `ui_options.json` 読み込みエラー（BOM関連） | ファイルをBOMなしUTF-8で保存し直す。v1.4.7以降はBOM混在に対応済み |
| ログが保存されない | `settings/pipeline_config.json` の `log_dir` パスが正しいか確認。フォルダが存在しない場合は自動作成されます |

### Qwen3系モデルの思考出力について

Qwen3.5などの思考系モデルは、プロンプト生成前に長い「思考過程」を出力することがあり、応答が遅くなったり余計なテキストが混入する場合があります。

**症状：**
- LLM生成ポジティブプロンプトに「The user wants me to generate...」のような文章が入っている
- 生成画像が真っ白・ノイズまみれになる
- 応答が極端に遅い

これらはQwen3系モデルの思考テキストがプロンプトに混入していることが原因です。以下のいずれかの方法で解決してください。

**LM Studioでの対処法（推奨）：**

LM Studioの右下「テンプレート」設定の先頭に以下を追記すると、思考出力が無効になります：

{% raw %}
```
{%- set enable_thinking = false %}
```
{% endraw %}

この設定はモデルレベルで思考を無効にするため最も確実です。

**システムプロンプトでの対処法：**

`settings/llm_system_prompt.txt` と `settings/preset_gen_prompt.txt` の冒頭に以下を追加することでも軽減できます：

```
/no_think
Answer immediately. Do not deliberate.
```

**llama.cpp系（LM Studio含む）の起動引数での対処法：**

```
--reasoning-budget 256
```

思考トークン数に上限を設けることで応答速度を改善できます。

### LLM APIエラーコード

| コード | 原因 | 対処 |
|--------|------|------|
| **400** Bad Request | モデル名が間違っている、またはAPIが対応していないパラメータを送信している | モデル名を確認。LM Studio以外ではMCPツール統合が原因になることも（設定でOFFに） |
| **401** Unauthorized | APIキーが未設定または無効 | Token欄にAPIキーを正しく入力して保存 |
| **404** Not Found | モデルが存在しない、またはURLが間違っている | モデル名・URLを再確認 |
| **429** Too Many Requests | レート制限に達した | しばらく待ってから再試行。無料枠の上限に達した場合は翌日リセットまで待つか、別モデルに切り替え |
| **500** / **503** | サーバー側の一時的なエラー | しばらく待ってから再試行 |

## ログ機能

v1.4.7以降、実行ログが自動的に保存されます。配布先での問題調査に活用できます。

### 保存場所とデフォルト設定

| 設定キー | デフォルト値 | 説明 |
|---------|------------|------|
| `log_dir` | `anima-pipeline/logs/` | ログ保存先フォルダ |
| `log_retention_days` | `30` | ログ保持日数（超過分は自動削除） |
| `log_level` | `normal` | `normal` または `debug` |

設定は `settings/pipeline_config.json` で変更できます。

### セキュリティ：秘匿情報のマスク

以下のキーワードを含む値はログ出力時に自動マスクされます。

- `token`
- `api key`
- `authorization`
- `bearer`

### 設定パネルのUI項目

設定パネル下部に以下の項目があります。

| 項目 | 説明 |
|------|------|
| LOG DIRECTORY | ログ保存先フォルダのパス |
| LOG RETENTION DAYS | ログ保持日数 |
| LOG LEVEL | ログレベル（normal / debug） |
| OPEN LOGS | ログフォルダをエクスプローラーで開く |
| EXPORT LOGS ZIP | ログをZIP圧縮してダウンロード |

## プリセットサムネイル一覧

v1.4.7以降、キャラプリセットをサムネイル付きで一覧表示できます。

### サムネイルの作成方法

1. 画像生成後、ギャラリー（生成履歴）でサムネイルにしたい画像を拡大表示
2. 「プリセットのサムネイル作成」ボタンをクリック
3. 保存先プリセットを選択して確定
4. サムネイルは `chara/<プリセット名>.webp` として保存されます

### 一覧からのキャラ追加

1. キャラセクションの「プリセット一覧」を展開
2. サムネイルをクリックしてプリセットを選択
3. 「更新先」セレクトで追加先のキャラ枠を選択
4. 「＋ キャラ追加」ボタンで、選択プリセットをセットした新しいキャラ枠を追加

> **注意：** 「＋ キャラ追加」はキャラ枠を追加して選択プリセットを **セット** するだけで、内容の読み込み（Load）は自動では行いません。読み込みは「読込」ボタンで別途実行してください。

- サムネイルは一覧を開いたときに取得（遅延読み込み）されるため、スマホでの初期表示が軽量化されています
- サムネイルが未作成のプリセットはグリッドに表示されません
