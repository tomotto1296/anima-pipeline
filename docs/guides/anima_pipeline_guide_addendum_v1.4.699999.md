# anima_pipeline guide addendum (v1.4.6 -> v1.4.699999)

## 目的
この追補は、v1.4.6以降で追加・変更された運用仕様を既存ガイドに補完するためのものです。

## 1. UI言語仕様

### 初期表示言語
- 起動時、OS言語を判定してUI言語を初期選択
  - 日本語OS: 日本語UI
  - 日本語以外: 英語UI

### 手動切替
- 画面上部の言語ボタンで切替可能
- ブラウザ保存領域に設定を保持

### 注意
- 一部文言は動的更新されるため、切替後に再描画される箇所まで含めて翻訳適用
- 翻訳作業中は `1.4.699x` 系で管理し、`1.4.7` へ上げない方針

## 2. コンソール表示言語
- コンソール表示は接続確認ブロック以降をOS言語基準に追従
- UI言語と完全同期ではなく、運用上はOS基準表示を優先

## 3. ログ機能

### 保存先
- 既定: `anima-pipeline/logs/`
- `settings/pipeline_config.json` で変更可能

### 設定項目
- `log_dir`
- `log_retention_days`（既定30日）
- `log_level`（`normal` / `debug`）

### 秘匿情報マスク
以下をログ出力時にマスク:
- token
- api key
- authorization
- bearer token

### 自動削除
- 保持期間（日数）を超えたログは自動削除

### API
- `GET /logs_info`
- `GET /logs_zip`

### UI
- LOG DIRECTORY
- LOG RETENTION DAYS
- LOG LEVEL
- OPEN LOGS
- EXPORT LOGS ZIP

## 4. 文字コード/設定読み込み
- UTF-8 BOM混在ファイルの読み込み耐性を強化
- `ui_options.json` などでBOM起因エラーが出るケースを修正

## 5. プリセット読み込み
- `000_default.json` が表示されない問題を修正
- `/chara_presets` 読み込みのエンコーディング互換を改善

## 6. プリセットサムネイル一覧（新規）

### 目的
- プリセットの視認性を上げ、キャラ追加までの操作を短縮する。

### 主な仕様
- キャラプリセットのサムネイル一覧UIを追加。
- ギャラリー画像を拡大表示した状態から「プリセットのサムネイル作成」を実行。
- 保存先は `chara/`、拡張子は `.webp` に統一。
- サムネイル取得は一覧展開時に実行（遅延読み込み）。
- スマホ表示はLoRAサムネイル一覧寄りのレイアウトに調整。
- 一覧選択プリセットは「キャラ追加」導線で反映（従来の即時読込とは別導線）。
- キャラ数は `0` を選択可能。

## 7. 運用ルール（更新）
- `releases/` は配布テスト環境として編集しない
- 不具合修正時は小刻みバージョン更新
  - 例: `1.4.6 -> 1.4.61 -> 1.4.62`

## 8. 変更ファイル（主要）
- `anima_pipeline.py`
- `settings/pipeline_config.default.json`
- `Update.md`
