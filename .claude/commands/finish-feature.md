実装完了時の最終確認を行います。

## Step 1: 変更内容の確認
以下を順番に実行して出力する:

```
git status
git diff main...HEAD --stat
git diff main...HEAD
```

## Step 2: スペック照合
現在のブランチ名からフィーチャーIDを推定し、対応する仕様書を `docs/specs/` から探す。

仕様書が見つかった場合、**完了条件（Acceptance Criteria）** と変更内容を照合し、
以下の形式でチェックリストを出力する:

```
## スペック照合結果

### ✅ 完了した項目
- ...

### ⬜ 未完了 / 未確認の項目
- ...

### ⚠️ スコープ外の変更（あれば）
- ...
```

## Step 3: 最終サマリー
以下を報告して終了する:
- 変更ファイル数とファイル名
- 追加/削除行数
- 未完了項目があれば明記

**プッシュ・PR作成は行わない。**
「プッシュしてください」と明示的に指示があった場合のみ `git push` を実行する。
