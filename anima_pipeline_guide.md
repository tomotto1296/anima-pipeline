# anima_pipeline 使い方ガイド

## 概要

ブラウザUIでキャラ・シーン情報を入力 → LLM（オプション）でDanbooruプロンプトを自動生成 → ComfyUI Animaワークフローへ自動送信するパイプラインツールです。

---

## 必要なもの

| ツール | バージョン | 用途 |
|--------|-----------|------|
| Python | 3.10以上 | サーバー起動 |
| ComfyUI | 0.16.4以上 | 画像生成 |
| Animaワークフロー | — | ComfyUI用ワークフロー |
| LLMサーバー（LM Studio等） | — | プロンプト自動生成（**オプション**） |

---

## ファイル構成

配布セット：

```
anima_pipeline/
  anima_pipeline.py              ← メインスクリプト
  start_anima_pipeline.bat       ← Windows起動用バッチ
  anima_pipeline_guide.md        ← このファイル
  image_anima_preview.json       ← Animaワークフロー（ComfyUI用）※後述
  settings/
    pipeline_config.json         ← 初期設定（ComfyUI URL・Node ID等）※APIキー入力後はgit管理除外推奨
    pipeline_config.default.json ← 初期値テンプレート（git管理用）
    ui_options.json              ← UIボタン選択肢の定義（編集可）
    llm_system_prompt.txt        ← LLMへのシステムプロンプト（編集可）
```

初回起動後に自動生成されるファイル（配布不要）：

```
  settings/
    pipeline_config.json         ← 設定変更後の上書き保存先（配布時の初期値から変更される）
    extra_tags.json              ← ポジティブExtraタグのプリセット
    extra_tags_negative.json     ← ネガティブExtraタグのプリセット
    style_tags.json              ← スタイルタグのプリセット
    anima_session_last.json      ← 最終セッションの自動保存
```

> **`image_anima_preview.json` について**  
> このファイルはリポジトリに含まれていません。以下の手順でご自身の環境から取得して `anima_pipeline.py` と同じフォルダに配置してください。
>
> 1. ComfyUIを起動
> 2. 右上のメニュー（三本線）→「Browse Templates」を開く
> 3. **Anima**テンプレートを選択して読み込む
> 4. メニュー →「Save (API Format)」で `image_anima_preview.json` として保存
> 5. 保存したファイルを `anima_pipeline.py` と同じフォルダに配置

## 起動方法

### Windows
`start_anima_pipeline.bat` をダブルクリック

### 手動起動
```bash
python anima_pipeline.py
```

起動後、ブラウザで `http://localhost:7860` を開く。

## 初回設定

画面上部の「▶ 設定」を開いて確認・入力します。設定は「💾 設定を保存」で `settings/pipeline_config.json` に保存されます。

> **②③④はAnima標準ワークフロー向けの初期値が最初から入っています。** 別のワークフローを使う場合のみ変更してください。

> **注意：** 初回起動時はプラットフォームが「なし」の状態です。LLMを使用する場合は設定パネルでプラットフォームを選択し、必ず「💾 設定を保存」を1回押してください。保存前に生成を実行するとプラットフォーム未設定のまま送信されます。

### 🔴 必須

| 番号 | 項目 | 初期値 | 説明 |
|------|------|--------|------|
| ① | ワークフローJSONパス | `image_anima_preview.json` | 同梱のワークフローを自動参照。別ワークフローはフルパスまたはファイル名で指定 |
| ② | Positive Node ID | `11` | Animaワークフロー標準。別ワークフローは要確認 |
| ③ | Negative Node ID | `12` | Animaワークフロー標準。別ワークフローは要確認 |
| ④ | ComfyUI URL | `http://127.0.0.1:8188` | 変更している場合のみ修正 |

### 🟠 LLMを使うなら必須

⑤ **LLMプラットフォーム**を選択するとURL・Token・モデル名の入力欄が開きます。

| プラットフォーム | 説明 |
|------|------|
| **なし** | LLM機能を使わない。テキスト直接入力のみ |
| **LM Studio** | ローカルLLM。無料・無制限だが高スペックPCが必要 |
| **Gemini** | Google Gemini API。無料枠あり（要APIキー、クレジットカード不要） |
| **カスタム** | 上記以外のOpenAI互換APIを手動設定 |

> **LLMプラットフォームの要件：** 使用するサービスがOpenAI互換API（`/v1/chat/completions`形式）に対応している必要があります。各サービスのAPI利用規約・料金・レート制限は各自でご確認ください。

| 番号 | 項目 | 説明 |
|------|------|------|
| ⑤-1 | LLM URL | APIのベースURL |
| ⑤-2 | LLM API Token | 認証が必要な場合のみ入力。ローカル（LM Studio）なら空欄でOK |
| ⑤-3 | LLMモデル名 | 使用するモデルの識別子 |

### 🟢 任意

| 番号 | 項目 | 説明 |
|------|------|------|
| ⑥ | LLMツール統合 | Danbooru RAG / Danbooru API / DuckDuckGoのON/OFF（LM Studio専用） |
| ⑦ | ComfyUI outputフォルダ | WebP変換用。**絶対パスで入力推奨**（例: `D:\ComfyUI_Portable\ComfyUI_windows_portable\ComfyUI\output`）。空欄の場合は自動推定を試みるが、ワークフローJSONをファイル名のみで指定している場合は失敗することがある |

## 💡 おすすめの使い方

**英語でタグを直接入力してLLMを介さずに送信するのが一番速いです。**

1. 「LLMを使用する」チェックをOFF
2. 詳細フィールドのLLMチェックをすべてOFF
3. 英語タグを直接入力して「▶ 生成開始」

LLMが便利なのは「キャラ名・作品名から自動でDanbooruタグを引っ張ってきたい」ときです。

### 入力欄の色分け

| 色 | 意味 |
|----|------|
| 🔵 水色 | 日本語OK（LLMが解釈して英語タグ化します） |
| 🟢 緑色 | 英語専用（Danbooruタグや英文を直接入力） |

---

## 別のワークフローで動かしたいとき

ComfyUIのAnima以外のワークフローや、別バリアントのワークフローで使う場合：

1. ComfyUIでワークフローを開き、**「Save (API Format)」でJSONを保存**
2. 設定の「① ワークフローJSONパス」を新しいJSONのパスに変更
3. 「② Positive Node ID」「③ Negative Node ID」をそのワークフローのノードIDに合わせて更新
4. 「💾 設定を保存」

> Node IDの確認方法：ComfyUIでプロンプトノードを右クリック →「Properties」のNode IDを確認

## 基本的な使い方

### A. キャラ情報を入力

**キャラ名・作品名**（水色欄・日本語OK）
- キャラ名と作品名を入力するとLLMがDanbooruで検索してタグを生成します
- **オリジナルキャラ**の場合は「オリジナル」ボタンを押す → 作品名欄が無効化され、LLMチェックが全部OFFになります
- キャラブロックの「＋」ボタンで複数キャラを同時設定可能

**性別・年齢**（`ui_options.json`で選択肢を変更可能）
- LLMチェックON: LLMが判断
- LLMチェックOFF: 選択値を直接Danbooruタグに変換（`1girl` / `1boy` など）

### B. 詳細を入力（任意）

「＋ 詳細」ボタンで展開します。

| フィールド | 説明 |
|-----------|------|
| ① 髪型 | 全体・前・後ろのグループから単一選択。自由入力欄は**日本語OK** |
| ② 髪色 | カラーボタンから選択 |
| ③ 目の状態 | 開閉（単一選択）・向き・状態のグループ |
| ④ 瞳の色 | カラーボタンから選択。オッドアイにも対応 |
| ⑤ 口 | 複数選択 |
| ⑥ 表情 | 複数選択 |
| ⑦ 肌の色 | ボタン選択 |
| ⑧ 衣装 | カテゴリ（全裸/半裸/全身/上下）を押すと色・種類が展開。上下は上半身・下半身を個別設定 |
| ⑨ バスト | 女性・不明キャラのみ表示 |
| ⑩〜⑫ | 背丈・体型・脚 |
| ⑬ 付属 | 耳・尻尾・翼（単一選択）、アクセサリー（複数選択） |
| ⑭ エフェクト | 汗/涙/オーラ/流血など複数選択 |
| ⑮ 動作・ポーズ | 姿勢/動作/腕・手/視線のグループから複数選択。自由入力は**英語タグ直接** |
| ⑯ 持ち物 | 「持つ」をONにすると展開。自由入力は**英語タグ直接** |
| ⑰〜⑲ | 画面上下・左右・カメラ |

**LLMチェックについて**
- ON → フィールドの値をLLMに渡してタグ生成を任せる
- OFF → 選択値をそのままDanbooruタグに直接変換（高速・確実）

### C. シーンを設定（任意）

| 項目 | 選択肢 |
|------|--------|
| 世界観 | 日常 / 和風 / 西洋 / 中華 / ファンタジー / SF / 廃墟 |
| 場所 | 屋外・屋内・特殊のカテゴリから選択、または自由入力（水色欄・**日本語OK**） |
| 時間帯 | 朝 / 昼 / 夕 / 夜 |
| 天気 | 晴 / 曇 / 雨 / 雪 |
| D. 補足メモ（日本語） | LLMへの雰囲気ヒント。**1回目の生成のみ**有効 |

### D. ポジティブプロンプト調整・追加（任意）

「▶ プロンプト調整・追加」欄。**再画像生成にも反映されます。**

| 番号 | 項目 | 説明 |
|------|------|------|
| ① | 期間タグ | Animaモデル向け最適化タグ（先頭挿入） |
| ② | 品質タグ | masterpiece / score_9 など |
| ③ | メタタグ | highres / absurdres など |
| ④ | 安全タグ | safe / nsfw など（単一選択） |
| ⑤ | スタイル | @アーティスト名。**プリセット登録可能**。右クリックで削除 |
| ⑥ | Extraタグ | プリセット登録可能。右クリックで削除。`settings/extra_tags.json`に永続保存 |
| ⑦ | 追記文（英語） | プロンプト末尾に直接追加する英文（緑色欄） |

### E. ネガティブプロンプト調整（任意）

「ネガティブプロンプト調整」欄（赤枠）。

| 番号 | 項目 | 備考 |
|------|------|------|
| ① | 期間タグ | ポジティブと共通 |
| ② | 品質タグ | 人間ベース: NORMAL/LOW/WORST（デフォルトON）、Pony（デフォルトOFF） |
| ③ | メタタグ | デフォルトOFF |
| ④ | 安全タグ | ポジティブと共通 |
| ⑤ | スタイル | ポジティブと共通 |
| ⑥ | Extraタグ | ネガティブ専用プリセット。`settings/extra_tags_negative.json`に永続保存 |
| ⑦ | 追記文（英語） | ネガティブプロンプト末尾に直接追加 |

### F. 生成

**LLMを使用する場合**
1. 「LLMを使用する」チェックがONであることを確認
2. 「▶ 生成開始」をクリック（またはCtrl+Enter）
3. LLMがプロンプトを生成 → ComfyUIに送信 → 画像生成

**LLMをスキップする場合**
1. 「LLMを使用する」チェックをOFF
2. フィールドのLLMチェックもすべてOFF推奨
3. 選択した値が直接ComfyUIに送られます（速い）

生成完了後、処理状況欄に以下が表示されます：
- **LLM生成ポジティブプロンプト** — LLMが生成した内容
- **ComfyUI 送信ポジティブプロンプト** — 最終的にComfyUIに送った内容（コピーボタン付き）
- **ComfyUI 送信ネガティブプロンプト** — ネガティブ側の送信内容（赤文字・コピーボタン付き）

## 再画像生成

「↻ 再画像生成」ボタンで、前回と同じプロンプトをComfyUIに再送信します。  
ポジティブ・ネガティブのプロンプト調整欄の変更は再生成にも反映されます。

---

## スタイルタグのプリセット管理

絵師スタイルを複数キャラ・複数セッションで使いまわせます。

1. スタイル入力欄（緑色）に`takeuchi naoko`などを入力して「追加」
2. プリセットボタンとして登録される（`settings/style_tags.json`に保存）
3. 次回以降はプリセットをクリックするだけでON/OFF切り替え
4. 右クリックでプリセットから削除

---

## 画像サイズのカスタマイズ

`settings/ui_options.json` の `image_size_presets` を編集すると、ドロップダウンの選択肢を変更できます。

```json
"image_size_presets": [
  {"v": "1024x1024", "label": "1024 × 1024　正方形"},
  {"v": "1920x1080", "label": "1920 × 1080　フルHD横"}
]
```

---

## WebP出力について

**注意：** WebP形式を選択した場合、ComfyUIが生成したPNGファイルを直後に自動でWebPに変換します。  
**変換後はPNG内に埋め込まれたプロンプト・ワークフロー情報は破棄されます。**  
ワークフロー情報を残したい場合はPNG形式を使用してください。

---

## セッション保存・読み込み

- 「💾 セッション保存」: 現在の全入力内容をJSONファイルとして保存
- 「📂 開く」: 保存したJSONを読み込み
- 最後の状態は `settings/anima_session_last.json` に自動保存（次回起動時に自動復元）

---

## LLMのシステムプロンプトのカスタマイズ

`settings/llm_system_prompt.txt` を直接編集することでLLMへの指示を変更できます。  
変更は次回の生成から即反映されます（py再起動不要）。

---

## UIボタンのカスタマイズ

`settings/ui_options.json` を編集すると、ほぼすべてのボタン選択肢を変更できます。  
変更はpyを再起動すると反映されます。

主な項目：
- `image_size_presets` — 画像サイズのプリセット
- `scene_world` / `scene_tod` / `scene_weather` — 世界観・時間帯・天気
- `gender_options` / `age_options` — 性別・年齢
- `quality_human` / `quality_pony` / `meta_tags` — 品質・メタタグとデフォルト選択
- `quality_human_neg` — ネガティブ品質タグ
- `outfit_data` — 衣装の色・種類
- その他多数

## トラブルシューティング

| 症状 | 確認事項 |
|------|---------|
| ComfyUIに送信されない | ComfyUI URLとワークフローJSONパスを確認 |
| Negative Promptが反映されない | 設定の「③ Negative Node ID」が正しく設定されているか確認 |
| LLMが応答しない | LLMサーバーが起動しているか確認。モデル名・URLが正しいか確認 |
| 画像が表示されない / WebP変換されない | ComfyUI outputフォルダに絶対パスを入力して保存。ワークフローJSONをファイル名のみで指定している場合は空欄での自動推定が失敗する |
| MCPツールが動かない | LLMサーバー側のMCPプラグインを確認。または設定欄でツール統合をOFFに |
| セッション復元がずれる | 古いセッションファイルを使っている場合はリロードで解消 |

### LLM APIエラーコード

| コード | 原因 | 対処 |
|--------|------|------|
| **400** Bad Request | モデル名が間違っている、またはAPIが対応していないパラメータを送信している | モデル名を確認。LM Studio以外ではMCPツール統合が原因になることも（設定でOFFに） |
| **401** Unauthorized | APIキーが未設定または無効 | Token欄にAPIキーを正しく入力して保存 |
| **404** Not Found | モデルが存在しない、またはURLが間違っている | モデル名・URLを再確認 |
| **429** Too Many Requests | レート制限に達した | しばらく待ってから再試行。無料枠の上限に達した場合は翌日リセットまで待つか、別モデルに切り替え |
| **500** / **503** | サーバー側の一時的なエラー | しばらく待ってから再試行 |
